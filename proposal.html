Choice #1: 3D LiDAR Mapper

I want to create a 3D mapper using a lidar sensor. The sensor would emit a laser pulse and then detect it using somesort of photodiode. The interest sensor would be on a gimbal that spins in two directions, ultimately creating a point map. I'm not sure of the scope of this project, but I could add some software that processes the point map and creates a 3D map/model. I know modeling engines like Blender can create 3d art via code. If I still had time, I could create a remote control car and use and train a neural network to drive it using the 3d lidar map Something like the video, but I would want the sensor to turn in two directions on a gimbal. Also, if I could create my own sensor, that would be ideal.

Choice #2: Kinetic Display

This project would create a machine that could arrange balls periodically along some sort of string. There would be, for instance, 100 strings in a 10x10 grid hanging vertically. The strings could move, and coupled with the machineâ€™s ability to arrange the balls at different lengths along the stings, the overall effect would be a 3d image. The balls could be attached to the stings via friction, but not so much that a claw or some sort of sphincter could change the ball's position. The string would then be moved via some pulley, moving the balls to the correct height. This is the coolest kinetic display I found, but another idea would be to create a screen using pistons or rotating tiles to create a non-light-based screen.
